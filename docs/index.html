CLI + MCP server · 100% local

One command generates
AI context files for every coding tool

Every AI assistant wants its own project file in its own format. claude-init scans your repo once and writes all 13 - no API key, no network.

$ npx @horiastanxd/claude-init
View on GitHub npm package

Why

Keeping a dozen context files in sync by hand is busywork, and they go stale the moment your stack changes.

One scan, every file

Stack, scripts, structure, env vars, conventions and git - detected once, written to all 13 formats in seconds.

100% local

No API key, no network, nothing leaves your machine. The optional --enrich flag is the only exception, and it is off by default.

Never goes stale

claude-init check diffs the generated files against disk and exits non-zero on drift. Drop it in CI or a pre-commit hook.

Also an MCP server

Run it over the Model Context Protocol so an agent can analyze a repo and write its own context files.

Usage

Zero config. Point it at a repo and go.

# generate everything in the current repo
npx @horiastanxd/claude-init

# only specific tools
npx @horiastanxd/claude-init -t claude,cursor

# verify files are up to date (CI / pre-commit)
npx @horiastanxd/claude-init check

# monorepo: generate into each workspace package too
npx @horiastanxd/claude-init --recurse

In CI with the action:

- uses: horiastanxd/claude-init@v1
  with:
    command: check
    targets: all

What it detects

Stack

Language, framework, runtime, package manager, database, test runner, build / monorepo tool, CI provider.

Languages

JS/TS (npm, pnpm, yarn, bun), Python (pip, uv, poetry), Rust, Go, Java (Maven, Gradle), Ruby, PHP.

Frameworks

Next.js, Remix, Astro, Angular, NestJS, SvelteKit, Nuxt, Vue, React, FastAPI, Django, Rails, Laravel, Spring Boot, and more.

Conventions

Strict mode, linter, formatter, import style, commit convention, env vars from .env.example, git authors and hot files.
